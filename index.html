Zero-to-SDDC: The Official Amazon EVS Deployment Guide

Amazon EVS: Step-by-Step Configuration Guide

March 25, 2026 | By David | Core Infrastructure & Automation

Welcome to the definitive manual guide for preparing your AWS account for **Amazon Elastic VMware Service (EVS)**. This post breaks down the "yesterday's build" into granular steps, ensuring you have every IP address, ASN, and Console path required for a zero-error deployment.

1 Phase 1: Capacity & IAM Prep

1.1 Bare Metal Quotas

Path: Console > Service Quotas > AWS Services > Amazon EC2

Search for "Running Dedicated i3.metal Hosts" (or i4i). Verify that the "Applied quota value" is at least 3. If it's 0, click "Request increase at account level" and specify your target region.

1.2 Service-Linked Role (SLR)

Path: Console > IAM > Roles > Create Role

Select "AWS Service", search for EVS, and select EVS - Elastic VMware Service. Click Next and Create. This gives EVS the permissions to manage VPC resources on your behalf.

2 Phase 2: VPC Underlay Construction

2.1 Create the VPC

Path: Console > VPC > Your VPCs > Create VPC

Path: Console > VPC > Subnets > Create Subnet

3 Phase 3: Route Server & BGP Peering

3.1 Provision VPC Route Server

Path: Console > VPC > Route Servers > Create Route Server

Enter ASN: 64512. Select evs-underlay-vpc. Once created, add two Endpoints in the evs-service-access-a subnet.

3.2 Configure BGP Peers (Neighbors)

Under your Route Server settings, navigate to Peers and add two neighbors corresponding to your NSX Edges:

Peer NameIP AddressASN
edge-0110.0.10.25165000
edge-0210.0.10.25265000

4 Phase 4: Route 53 DNS & Resolver

4.1 Private Hosted Zones

Path: Console > Route 53 > Hosted Zones > Create

Create the following three zones and associate them with evs-underlay-vpc:

  • Forward: evstest.com
  • Reverse MGMT: 10.0.10.in-addr.arpa
  • Reverse Host: 11.0.10.in-addr.arpa

4.2 Inbound Resolver Endpoints

Path: Console > Route 53 > Resolver > Inbound Endpoints > Create

Select evs-underlay-vpc. Assign a Security Group that allows UDP/TCP port 53. Under IP Addresses, manually assign:

  • IP 1: 10.0.1.100 (Subnet: evs-service-access-a)
  • IP 2: 10.0.1.110 (Subnet: evs-service-access-a)
Important: Ensure you create A and PTR records for your core components (vCenter, SDDC Manager, NSX Managers) and your ESXi hosts (e.g., esxi-01 to esxi-04) within these zones.

5 Phase 5: DHCP Option Set Association

5.1 Create DHCP Option Set

Path: Console > VPC > DHCP Option Sets > Create

FieldValue
Domain Nameevstest.com
Domain Name Servers10.0.1.100, 10.0.1.110
NTP Servers169.254.169.123

Finally, go back to Your VPCs, select your VPC, click Actions > Edit VPC Settings, and change the DHCP Option Set to the one you just created.

Master Automation: Terraform Template

If you prefer to deploy this entire stack in 60 seconds, use our production-grade Terraform configuration.

main.tf (Comprehensive) Version 1.2 
# VPC & Networking
resource "aws_vpc" "evs_vpc" {
  cidr_block = "10.0.0.0/16"
  enable_dns_hostnames = true
  enable_dns_support = true
  tags = { Name = "evs-vpc" }
}

resource "aws_subnet" "service_access" {
  vpc_id = aws_vpc.evs_vpc.id
  cidr_block = "10.0.1.0/24"
  availability_zone = "us-west-2a"
}

# Route 53 Inbound Resolver
resource "aws_route53_resolver_endpoint" "evs_dns" {
  name = "evs-dns-resolver"
  direction = "INBOUND"
  security_group_ids = [aws_security_group.dns_sg.id]

  ip_address { subnet_id = aws_subnet.service_access.id; ip = "10.0.1.100" }
  ip_address { subnet_id = aws_subnet.service_access.id; ip = "10.0.1.110" }
}

# VPC Route Server
resource "aws_vpc_route_server" "evs_rs" {
  amazon_side_asn = 64512
  tags = { Name = "evs-route-server" }
}

resource "aws_vpc_route_server_peer" "edge_01" {
  route_server_id = aws_vpc_route_server.evs_rs.route_server_id
  peer_address = "10.0.10.251"
  bgp_options { peer_asn = 65000 }
}
